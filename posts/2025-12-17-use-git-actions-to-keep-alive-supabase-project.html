使用 Git Actions 给 Supabase 项目做“保活”

📅 2025/12/17 ✍️ 字数：2517
日更挑战最佳实践Git ActionsSupabaseSupabase 保活

前言

我的博客系统后台的数据库的项目是放在supabase，但是supabase免费版本，如果你很久没写博客，或者这个项目属于处在静默状态，那么supabase就会回收这个项目的资源，大概是一周左右，整个项目就会失去active状态，甚至是永久性的回收。所以如何最低限度地做保活就很重要了。

如何定义保活

官方文档其实并没有明确规定什么时间点会回收你的数据库资源，造成它的原因可能是不定的，有可能是账单问题，使用了付费功能，长期不使用的项目等待。对我来说，或者说对免费用户来说，可能长期不使用项目是有可能得。 既然没有明确规定，那么如何保活就可能是那不太准的事，需要结合实际来。结合收集的资料来看，要有比较平凡数据库查询调用，特别是重要业务数据库的连接调用,是能够保活的关键。在这样的背景下，我们来做保活。

总结
