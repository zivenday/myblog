Nestjs是用来做什么？

说到Nestjs，首先想到的是，它是干嘛的？做后台的？

那如果要装一下的话，会怎么回答？

用于构建服务器端应用的 Node.js 框架。

这个回答了它是什么，没有回答用来做什么的，做后台的包括两个，一个是做后台API，一个是服务器。

主要用来开发 后端 API、Web 服务和微服务。

后端API，你会想到什么？

想不到？就是restfull api和接口

restfullapi是什么，它和普通接口有什么区别？

全称是资源的表现形式（JSON / XML）状态转移api，它有一套风格，规范，它和普通接口的差别比如，它吧服务器数据当成某种资源。

有了restfull api 为什么还要普通接口那样？

restfull 非常适合增删改查，前面说过，数据被看成资源获取的感觉的类型。增删改查，四种状态。 但是还要很多接口不是增删改成而已，有具体的业务，比如登录，支付，导出等，如果这时候还用restfull，接口会变得很难看懂。普通接口里面可能涉及多个底层操作，这时候，这个包裹起来的函数用restfull，就不是增删改查了。

Nestjs做微服务，是什么样子的？

就是一个个，小小的服务，各自独立，各自可以不同技术栈，部署的地方不同，彼此之间通过消息或者RPC通信。消息就是像发邮件，通过中间件实现，RPC就是通过网络接口实时通信，类似打电话。

API 网关是什么呢？它长什么样子？

网关的工作是转发请求到对应的服务，比如说ngx是专门这么做这个,有点像红绿灯来指明方向路由。API 网关除此之外，还有鉴权，限流一些通用的功能，分给各个微服务。

网关的实现两者，项目自身实现，第二有专门的框架Nginx
