使用 Git Actions 给 Supabase 项目做“保活”

📅 2025/12/17
日更挑战最佳实践Git ActionsSupabaseSupabase 保活

前言

我的博客系统后台的数据库的项目是放在supabase，但是supabase免费版本，如果你很久没写博客，或者这个项目属于处在静默状态，那么supabase就会回收这个项目的资源，大概是一周左右，整个项目就会失去active状态，甚至是永久性的回收。所以如何最低限度地做保活就很重要了。

如何定义保活

官方文档其实并没有明确规定什么时间点会回收你的数据库资源，造成它的原因可能是不定的，有可能是账单问题，使用了付费功能，长期不使用的项目等待。对我来说，或者说对免费用户来说，可能长期不使用项目是有可能得。 既然没有明确规定，那么如何保活就可能是那不太准的事，需要结合实际来。结合收集的资料来看，要有比较平凡数据库查询调用，特别是重要业务数据库的连接调用,是能够保活的关键。在这样的背景下，我们来做保活。

结合Github Actions WorkFlow

第一步

我们使用定义keep-supabase-awake.yml脚本放到.github/workflows文件夹之下，让git Action去调用这个脚本，每小时，或者每天去调用数据库查询。git action 也可以手动操作调用脚本。

第二步

可以看到supabase 这里 有个直连方式

直连方式，后面折叠的内容却有提示像Github Actions ipv4-only 是不适合直连的，要用pooler

所以要复制pooler这个选项下面这个链接来连。

第三步 yml脚本

脚本如下：

name: Keep Supabase Awake


on:

schedule:

# Every hour at minute 0 (UTC)

- cron: '0 * * * *'

# Allow manual runs for testing

workflow_dispatch: {}

  

concurrency:

group: keep-supabase-awake

cancel-in-progress: false

  

jobs:

ping-db:

runs-on: ubuntu-latest

timeout-minutes: 5

env:

# Avoid long waits on failed connections

PGCONNECT_TIMEOUT: '30'

steps:

- name: Install psql client

run: |

sudo apt-get update -y

sudo apt-get install -y postgresql-client

  

- name: Run lightweight SQL (pooler via URI)

env:

DATABASE_URL: ${{ secrets.DATABASE_URL }}

run: |

if [ -z "$DATABASE_URL" ]; then

echo "Missing DATABASE_URL secret." >&2

exit 1

fi

  
# Ensure sslmode=require present

URL="$DATABASE_URL"

case "$URL" in

*"sslmode="*) URL_WITH_SSL="$URL" ;;

*"?"*) URL_WITH_SSL="${URL}&sslmode=require" ;;

*) URL_WITH_SSL="${URL}?sslmode=require" ;;

esac

  

# Try up to 3 times via pooler URI (query keep-alive table)

for i in 1 2 3; do

echo "Attempt $i via pooler URI: SELECT count(*) FROM \"keep-alive\";"

psql "$URL_WITH_SSL" -v "ON_ERROR_STOP=1" -c "SET application_name='gha-keepalive'; SELECT count(*) FROM \"keep-alive\";" && exit 0

echo "Attempt $i failed. Sleeping 10s..."

sleep 10

done

echo "All attempts failed." >&2

exit 1

里面的DATABASE_URL就是刚刚说的链接，并且要在GitHub Actions 里创建这个参数保存

第四步

创建keepAlive 数据库，作为保护的数据库查询

CREATE TABLE "keep-alive" (

id BIGINT generated BY DEFAULT AS IDENTITY,

name text NULL,

created_at timestamptz DEFAULT now(),

CONSTRAINT "keep-alive_pkey" PRIMARY KEY (id)

);

  

INSERT INTO "keep-alive" (name) VALUES ('ping');

接下来就可以直接去跑了

那么能在supabase看到是否跑成功了呢？

在性能观察这里其实可以侧面证实，已经跑成功了

接下来，就看看，你的项目会不会保活成功了。因为很多都是未知数，还是要看实践，还有可能跟项目本身有关。

总结

Supabase 免费版在项目长期处于静默状态时，可能会回收资源甚至永久删除项目。为了确保数据库的持续活跃，以上提出了一种低成本、自动化的方案，结合 GitHub Actions 和 Supabase 连接池，通过定时执行轻量级数据库查询来维持项目活跃状态。当然这一切还与项目本身特性有关，但是万变不如其中。
